# 点九图（9-Patch）测试工具

一个纯前端、可离线运行的 Android 9-Patch（点九图）测试工具。

你可以直接导入 `.9.png`，自动解析四边 1px 标注线，实时预览不同尺寸下的拉伸效果，查看内容区（padding）与 stretch 分段，并支持导出渲染结果或重新导出修改后的 `.9.png`。

## 功能特性

- 支持点击选择或拖拽导入 `.9.png`
- 自动解析：
  - 顶边：水平 stretch 区
  - 左边：垂直 stretch 区
  - 底边：内容区 X
  - 右边：内容区 Y
- 自动去除 1px 标注边并生成实际渲染图
- 多尺寸实时预览，便于验证适配效果
- 支持显示/隐藏：
  - 内容示例文字
  - 内容区叠加
  - stretch 分段叠加
  - 图片阴影
  - 容器裁剪圆角
- 支持修改内容示例文本
- 支持通过“标注编辑（单段）”调整四周标注线并实时验证
- 支持导出：
  - 当前尺寸渲染 PNG
  - 修改标注后的 `.9.png`

## 适用场景

- Android 开发：验证点九图在不同宽高下是否正常拉伸
- 设计自检：检查导出的 `.9.png` 标注是否正确
- 协作沟通：快速定位内容不居中、内容区过小、拉伸区域不合理等问题

## 使用方式

### 1. 打开项目

直接双击打开 `index.html`，或者使用任意静态服务器。

例如：

```bash
python3 -m http.server 5173
```

然后访问：

```text
http://localhost:5173/
```

### 2. 导入点九图

- 点击“选择或拖入 .9.png”
- 或直接拖拽图片到上传区域
- 也可以使用“内置示例”快速体验功能

### 3. 调整预览

- 设置目标宽 / 高
- 修改“内容示例”文本
- 切换背景模式：棋盘格 / 深色 / 浅色
- 打开或关闭叠加项，观察内容区与 stretch 行为

### 4. 编辑标注线

左侧“标注编辑（单段）”区域可直接调整四边标注：

- `Stretch X（顶边）`
- `Stretch Y（左边）`
- `内容区 X（底边）`
- `内容区 Y（右边）`

支持两种方式：

- 输入数字
- 拖动滑杆
- 在编辑预览图上拖动四周的小手柄

修改后右侧预览会实时刷新。

### 5. 导出

- `导出当前 PNG`：导出当前目标尺寸下的渲染结果
- `导出 .9.png`：导出修改标注线后的点九图文件

## 交互说明

### 内容区为什么重要

点九图不只决定“哪里可以拉伸”，还决定“内容应该放在哪里”。

如果文字或内容看起来不居中，通常不是渲染出错，而是：

- 底边内容区标注不合理
- 右边内容区标注不合理
- 点九图本身视觉中心与几何中心不一致

这也是项目里加入“标注编辑（单段）”的原因：可以边调边看，快速找到更合适的内容区。

### 关于容器裁剪圆角

默认情况下，工具不会再用外层容器强行裁剪图片，避免把点九图真实透明边缘或外轮廓截掉，导致预览失真。

如果你需要卡片式展示效果，可以手动打开“容器裁剪圆角”。

## 技术实现

- 纯静态页面：`HTML + CSS + 原生 JavaScript`
- 图像处理与渲染：`Canvas 2D`
- Nine-Patch 渲染方式：按 stretch 分段切片后分配到目标尺寸，再逐块 `drawImage`

核心流程：

1. 读取原始 `.9.png`
2. 解析四边黑色标注像素
3. 去除 1px 外边框得到实际图像
4. 构建横向 / 纵向切片
5. 依据目标尺寸重新分配 stretch 区宽高
6. 绘制最终预览

## 已知限制

- 当前“标注编辑”按单段区间处理，更适合常见按钮、气泡、卡片等场景
- 多段 stretch 的原始解析与渲染是支持的，但编辑器目前会合并为单段来调整
- 对不符合 9-Patch 规范的图片，会按兜底逻辑处理，结果仅供参考
